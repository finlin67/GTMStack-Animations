# `/gallery` Integration Guide (Two Audiences)

This single document explains how the `/gallery` experience in **GTMStack_pro_production** is populated by content from **GTMStack_Animations**.

It is written for:
- **Developers** (handoff + troubleshooting)
- **Non-technical users** (navigation, testing, basic troubleshooting)

---

## 1) Purpose of the Animation Gallery

The `/gallery` page is the browsing and preview surface for animation assets maintained in the `GTMStack_Animations` repository.

In practice, the gallery is meant to:
- Provide a browsable catalog of animation projects
- Let users filter by content groups (especially category/tag-based groupings)
- Show visual thumbnails and titles in a card grid
- Open a modal/pop-up with a richer preview and metadata
- Link back to source/docs (GitHub project + README links when available)

The gallery exists so animation content can be managed in one source repo (this repo), then consumed in the production website repo.

---

## 2) What Populates the Gallery UI

## 2.1 Source of truth

From this repo, the main source artifact is:
- `exports/gallery-manifest.json`

This is generated by:
- `scripts/export-gallery-manifest.js`

And includes fields used by website rendering, including:
- `id`, `slug`
- `animationId` (optional mapping to website React registry)
- `title`
- `category`
- `projectPath`
- `entryHtml`, `entryType`
- `readmePath`
- `summary`
- `tags`
- `thumbnailPath`
- `githubUrl`, `githubReadmeUrl`
- `updatedAt`

## 2.2 Gallery template and grid rendering (website repo)

In `GTMStack_pro_production`, audit references show this path:
- Template/page layer: `src/templates/gallery/GalleryMainTemplate.client.tsx`
- Grid shell/cards: `components/gallery/StitchGalleryShell.client.tsx`

These consume manifest-backed gallery items and render:
- Thumbnail image (from `thumbnailPath` → website public URL)
- Card title (from `title`)
- Category/tag context (used for filtering/search/display)

## 2.3 Sidebar navigation filters

Sidebar filters are populated from manifest metadata, primarily:
- `category` (primary grouping)
- `tags` (secondary refinements, where implemented)

So if categories/tags are wrong in manifest data, sidebar filtering will look wrong even when UI code is correct.

## 2.4 Thumbnails

Thumbnails are generated in this repo by:
- `scripts/generate-thumbnails.js`
- npm scripts: `npm run thumbs:dry` and `npm run thumbs:gen`

Thumbnail sources:
- Project-local `preview.png` in each animation folder
- Optional central output in `assets/thumbnails/`

The manifest exporter resolves `thumbnailPath` from those files, then website sync copies them into website public image paths.

## 2.5 Modal (pop-up) and content inside it

Audit references in `GTMStack_pro_production` identify modal and mapping files:
- Modal UI: `components/gallery/GalleryModal.tsx`
- Manifest loader: `src/lib/galleryManifest.ts`
- ID resolver map: `src/lib/galleryAnimationMap.ts`
- Registry source: `src/data/animations.ts` (`ANIMATION_REGISTRY`)

Modal preview content is typically chosen from:
1. A mapped React animation component (via `animationId`/resolver)
2. Fallback iframe source from `entryHtml`
3. Thumbnail fallback + “preview unavailable” style state

The modal’s textual content comes from manifest metadata (title, summary, category/tags, and available links such as GitHub/README).

## 3) End-to-End Process: How This Repo Populates `/gallery` in GTMStack_pro_production

## 3.1 Content authoring in `GTMStack_Animations`

Animation projects live under:
- `animations/<category>/<slug>/...` (or some flat legacy folders)

Typical inputs per project:
- Entry file (`*.html` and/or `*.tsx`/`*.ts`)
- `README.md` (title + summary extraction source)
- `preview.png`/`thumbnail.*` assets
- Optional `meta.json` (content model validation path)

## 3.2 Generate/refresh thumbnails

Commands:
```bash
npm run thumbs:dry
npm run thumbs:gen -- --central-output=assets/thumbnails
```

Primary file:
- `scripts/generate-thumbnails.js`

Output/report:
- Per-project `preview.png`
- Optional central files in `assets/thumbnails/`
- `reports/thumbnail-report.md`

## 3.3 Generate/refresh gallery manifest

Commands:
```bash
npm run manifest:dry
npm run manifest:gen
```

Primary file:
- `scripts/export-gallery-manifest.js`

Output/report:
- `exports/gallery-manifest.json`
- `reports/gallery-manifest-report.md`

## 3.4 (Optional) scaffold `preview.html` for TSX-only folders

Commands:
```bash
npm run preview-html:dry
npm run preview-html:gen
```

Primary file:
- `scripts/scaffold-preview-html.js`

This helps thumbnail generation discover TSX-only projects, because the screenshot workflow captures `.html` entries.

## 3.5 Sync to website repo (`GTMStack_pro_production`)

In the website repo, run (per audit/runbook):
```bash
node scripts/sync-gallery.js --apply --copy-entry-html
```

Expected result in website repo:
- Manifest copied to `src/data/gallery-manifest.json`
- Thumbnails copied to `public/images/...`
- HTML entry assets copied to `public/animations/...` (when `--copy-entry-html` is used)

This sync step is what makes newly generated animation data visible on `/gallery`.

## 4) Developer Handoff + Troubleshooting Guide

## 4.1 Quick “contract” checklist

If `/gallery` is wrong, verify in this order:
1. `exports/gallery-manifest.json` has correct item metadata
2. `thumbnailPath` files actually exist in this repo
3. Website sync ran successfully (manifest + thumbnails + entry HTML)
4. Website resolver can map IDs to registry components (or fallback iframe works)

## 4.2 Key files in **this** repo

- `scripts/export-gallery-manifest.js` (manifest generation logic)
- `scripts/generate-thumbnails.js` (screenshot generation)
- `scripts/scaffold-preview-html.js` (TSX-to-preview HTML helper)
- `exports/gallery-manifest.json` (integration payload)
- `reports/gallery-manifest-report.md` (manifest diagnostics)
- `reports/thumbnail-report.md` (thumbnail diagnostics)
- `assets/thumbnails/` (optional centralized thumbnail artifacts)

## 4.3 Common failure patterns and fixes

### A) Missing cards / no data in gallery
- Cause: stale or missing synced manifest in website repo
- Fix: regenerate manifest in this repo, then re-run website sync

### B) Filters wrong/incomplete
- Cause: bad `category`/`tags` values in manifest records
- Fix: correct source project metadata (README-derived or manifest logic), regenerate + sync

### C) Thumbnails wrong or text-card placeholders
- Cause: screenshot captured scaffold placeholder HTML
- Fix: replace placeholder preview source with true render source; regenerate thumbnails; sync again

### D) Modal opens but preview is blank
- Cause: ID map misses registry entry and iframe HTML not copied
- Fix: in website repo, ensure sync uses `--copy-entry-html`; confirm modal fallback precedence/mapping

### E) Duplicate-looking items
- Cause: same base slug in multiple categories or versions (`-v2`, `-v3`)
- Fix: enforce canonical/version policy and/or deprecate duplicates in manifest strategy

## 4.4 Recommended repeatable pipeline

From this repo:
```bash
npm run manifest:gen
npm run preview-html:gen
npm run thumbs:gen -- --central-output=assets/thumbnails
npm run manifest:gen
```

Then in website repo:
```bash
node scripts/sync-gallery.js --apply --copy-entry-html
```

## 5) Non-Technical User Guide (Navigate, Test, Basic Troubleshooting)

## 5.1 How to use `/gallery`

1. Open `/gallery` on the website.
2. Use the left sidebar to filter by category/tag.
3. In the card grid, check thumbnail + title.
4. Click a card to open the pop-up/modal.
5. In the modal, confirm preview loads and information is readable.

## 5.2 What “good” looks like

- Cards show meaningful thumbnail images (not generic placeholders)
- Titles match the content shown
- Filters narrow results correctly
- Modal opens quickly and shows animation preview
- Links (if shown) open expected source/readme pages

## 5.3 Basic troubleshooting (no code changes)

If something looks wrong, try:
1. Hard refresh browser (`Ctrl+Shift+R` on Windows)
2. Re-open modal for a different card, then return
3. Test one item from another category (to check if issue is item-specific)

Report issues with these details:
- Page URL
- Card title
- What happened (e.g., wrong thumbnail, blank modal, broken link)
- Screenshot if possible
- Time/date of test

## 5.4 What to escalate to a developer

Escalate when you see:
- Large groups missing from gallery
- Sidebar filters not changing results
- Many blank modals
- Repeated broken thumbnails after hard refresh

Ask developer to “regenerate manifest + thumbnails and re-sync gallery assets from GTMStack_Animations.”
